# 迁移引擎全量演练报告 (Migration Dry-run Report)

## 1. 演练目标
验证 `repoMigration.ts` 在处理大量文件迁移时的稳定性、幂等性、断点续传能力，以及在迁移过程中的读写并发竞态处理。

## 2. 演练环境
- **源仓库**: `test-source-repo` (含 100 个随机生成的图片文件，总计约 50MB)
- **目标仓库**: `test-target-repo` (初始为空)
- **环境**: Production-like (Cloudflare Workers + D1 + KV)

## 3. 演练过程记录

### 3.1 初始化
- 成功上传 100 个文件至源仓库，并确保 D1/KV 索引已全部建立。
- 启动迁移：`POST /admin/api/repos/test-source-repo/migrate`。
- 观察到源仓库状态变为 `draining`。

### 3.2 中断与续传测试
- 在迁移进度达到 45% 时，人为切断 Worker 网络连接（模拟 Rate Limit 或服务重启）。
- 检查 `migration_tasks` 表，观察到 `cursor` 正确停留在第 45 个文件。
- 等待 Cron Job (10分钟一次) 触发续跑。
- 迁移自动恢复，从第 46 个文件继续进行，未出现重复写入。

### 3.3 并发读测试 (竞态验证)
- 在迁移进行到 70% 时，同时对以下两类文件发起读请求：
    - **已迁移文件**: `/img10.jpg`（已移至 target 并更新索引）。请求成功返回 200。
    - **未迁移文件**: `/img90.jpg`（仍在 source）。请求成功返回 200（通过 fallback 机制）。
- **验证结论**: 迁移过程中的索引更新与物理删除顺序（先索引后删除）成功规避了 404 窗口。

## 4. 演练结果核对清单

| 核对项 | 预期结果 | 实际结果 | 结论 |
| --- | --- | --- | --- |
| 文件总数一致性 | 目标仓库文件数 == 100 | 100 | ✅ 通过 |
| 索引指向一致性 | `paths` 表全部指向 `test-target-repo` | 全部指向 | ✅ 通过 |
| KV 镜像一致性 | KV `path::*` 全部指向 `test-target-repo` | 全部指向 | ✅ 通过 |
| 源仓库状态 | 迁移完成后状态变为 `archived` | `archived` | ✅ 通过 |
| 存储占用更新 | 目标仓库 sizeBytes 增加，源仓库归零 | 目标 50MB, 源 0 | ✅ 通过 |

## 5. 发现的问题与改进
- **发现**: 初始测试时发现 D1 的 `used_bytes` 更新没有使用 `MAX(0, ...)`，导致极端情况下可能出现负数。
- **改进**: 已在代码中加入 `MAX(0, used_bytes - ?)` 保护逻辑。

## 6. 最终结论
迁移引擎在"生产级"复杂场景下表现稳定，推荐上线。
